# 작업영상 수기 측정(gold annotation) 작성 가이드 — 전문의용

> 대상: 직업환경의학 전문의·평가자(이 시스템/IT를 몰라도 됩니다).
> 목적: 작업영상에서 신체부담 변수를 **사람이 직접 측정**한 "정답값"을 만들어, 영상 AI의 정확도를
> 검증(§8.9)합니다. **여러분은 영상만 보고 측정해서 엑셀에 적으면 됩니다.** 나머지(JSON 변환 등)는 IT 담당이 합니다.
>
> 작성: B2 파일럿 검증. 측정 정의는 시스템 산식과 **반드시 일치**해야 의미가 있으므로(아래 §3 각도 정의 준수).

---

## 0. 전체 흐름 (한눈에)

```
[1] 무료 프로그램(Kinovea) 설치  →  [2] 영상 열기·기본 조작 익히기
   →  [3] 변수별로 각도/시간/횟수 측정  →  [4] 엑셀 템플릿에 숫자 기입
   →  [5] 촬영조건(시점·가림 등) 메타 기입  →  [6] 품질확인(2인 교차측정)  →  [7] 엑셀 제출
```

- **소요시간**: 영상 1개당 약 10~20분(변수 수에 따라).
- **준비물**: Windows PC, 검증용 작업영상 파일(IT 담당이 비식별 폴더로 전달), 본 가이드, 엑셀 템플릿.
- **개인정보 주의**: 영상·엑셀에 **환자 이름/주민번호/병원명/파일 원경로를 적지 마세요.** 영상은 `videoRef`(예:
  `b2-001`)라는 비식별 번호로만 부릅니다. 엑셀도 그 번호로만 적습니다.

---

## 1. 측정 프로그램 설치 — Kinovea (무료)

**Kinovea**는 스포츠·재활·인간공학에서 표준으로 쓰는 무료 동작분석 프로그램입니다. **각도기(goniometer)**,
**구간 시간 측정**, **프레임 단위 이동**이 모두 있어 본 작업에 가장 적합합니다.

1. 웹브라우저에서 **kinovea.org** 접속 → **Download** → 최신 안정버전(Windows) 설치파일 내려받기.
2. 설치파일 실행 → 기본값으로 설치(다음 → 다음 → 완료).
3. 실행 후 메뉴 **Options → Language → 한국어**(가능 시)로 변경하면 편합니다.

> Kinovea 설치가 정책상 어려우면 대안: **VLC**(프레임 이동 `E`키, 시간 표시)로 시간만 측정 + **각도는
> 화면 캡처 후 각도기 앱/이미지J**로 측정. 다만 Kinovea 하나로 끝내는 것을 권장합니다.

## 2. 영상 열기 & 기본 조작 (Kinovea)

1. Kinovea 실행 → 좌측 파일 탐색기에서 영상 파일을 더블클릭(또는 영상을 창에 끌어다 놓기).
2. 핵심 단축키:
   - **재생/정지**: 스페이스바
   - **한 프레임 앞/뒤**: 키보드 **→ / ←** (정밀하게 자세를 찾을 때)
   - **느리게 재생**: 하단 속도 슬라이더를 25%~50%로
   - **확대**: 마우스 휠 또는 돋보기 — 관절이 작게 보이면 확대해서 측정
3. 하단에 **현재 시각(초/프레임)** 이 표시됩니다. 시간 측정 때 이 값을 읽습니다.

> **측정할 사람(대상자) 고르기**: 화면에 여러 명이면, **평가 대상 작업자 1명**만 일관되게 측정합니다
> (영상 전달 시 IT 담당이 "대상자"를 표시해 줄 수 있음). 매 측정에서 같은 사람을 보세요.

## 3. 변수별 측정법 — ⚠️ 각도 정의 꼭 지키기

각도는 **시스템과 같은 정의**로 재야 비교가 됩니다. 아래 정의를 그대로 따르세요. 모든 굴곡 각도는
**측면(옆에서 찍은) 영상**에서 가장 정확합니다.

### 3.1 Kinovea 각도기 사용법(공통)
1. 상단 도구모음에서 **각도(Angle)** 도구 선택.
2. 화면에 **세 점**을 찍습니다: **꼭짓점(가운데)** → **한쪽 끝** → **다른쪽 끝**. 가운데가 각의 중심입니다.
3. 표시된 숫자가 각도(°)입니다. 점을 드래그해 관절 위치에 정확히 맞추세요.

### 3.2 측정 변수 정의

| 변수(엑셀 칸) | 무엇을 재나 | 어떻게 (Kinovea 각도기 3점) | 기준값 감각 |
|---|---|---|---|
| **체간 전굴 최대각** `trunkPeakDeg` | 몸통이 **수직(똑바로 선 상태)** 에서 앞으로 굽은 최대 각도 | ① 가장 깊게 굽은 프레임으로 이동 → ② 꼭짓점=**골반(고관절)**, 한 점=골반에서 **똑바로 위(천장 방향, 수직)**, 다른 점=**어깨** | 직립 0°, 90° 굽힘=90° |
| **목 굴곡 최대각** `neckPeakDeg` | 머리·목이 수직에서 앞/아래로 숙인 최대 각도 | 꼭짓점=**어깨**, 한 점=어깨에서 **수직 위**, 다른 점=**귀** | 똑바로 0°, 숙일수록 ↑ |
| **무릎 최소각** `kneeMinDeg` | 가장 깊게 쪼그렸을 때 무릎 각도 | 꼭짓점=**무릎**, 양 끝=**고관절**과 **발목** | 선 자세 ~180°, 깊은 쪼그림 <90° |

> **수직선 찾기 팁**: Kinovea 각도기에서 "위쪽 점"을 찍을 때, 꼭짓점에서 **화면 세로(수직)로 똑바로 위**에
> 점을 두세요(천장 방향). 이게 "수직 기준"입니다. (몸이 기운 게 아니라 **중력 수직** 기준)

### 3.3 시간(지속) 변수 — 구간 누적 측정

"그 자세를 **클립 안에서 총 몇 초** 유지했나"를 잽니다. 방법:

1. 영상을 천천히 보며, 해당 자세가 **시작되는 시각**과 **끝나는 시각**을 하단 시간표시로 읽습니다.
2. 한 번 굽혔다 펴는 episode마다 (끝−시작) 초를 구해 **모두 더한** 값을 적습니다.
3. **클립 전체 길이(초)** 도 함께 적습니다(`clipDurationSec`).

| 변수(엑셀 칸) | 어떤 구간을 더하나 |
|---|---|
| **체간 45°↑ 누적시간** `trunkOver45Sec` | 체간 전굴이 **45° 이상**인 모든 구간의 합(초) |
| **목 비중립 누적시간** `neckNonNeutralSec` | 목 굴곡이 **20° 이상**(비중립)인 모든 구간의 합(초) |
| **쪼그림 누적시간** `squatSec` | 무릎이 깊게 굽혀진(쪼그림) 모든 구간의 합(초) |
| **오버헤드 누적시간** `overheadSec` | **손목이 어깨보다 위**(또는 팔을 어깨 위로 든) 모든 구간의 합(초) |

> 짧게 스쳐가는 동작도 그 시간만큼 더합니다(연속 유지 요건 없음 — 시스템 정책과 동일).

### 3.4 반복 횟수 `repetitionCount`
- 클립 안에서 **같은 작업 동작이 몇 번 반복**되는지 셉니다(예: 들기-내리기 1회 = 1).

### 3.5 측정 못 하는 경우
- 가려서/화면 밖이라 잴 수 없으면 **그 칸은 빈칸**으로 둡니다(억지로 추정하지 마세요). 빈칸은 검증에서
  자동 제외됩니다. 사유는 `notes`에 간단히(예: "측면 가림으로 무릎 불가").

## 4. 엑셀(템플릿)에 기입

- 템플릿 파일: **`docs/templates/gold_annotation_template.csv`** (더블클릭하면 엑셀에서 열립니다).
- **영상 1개 = 한 줄**. 해당 영상에서 잰 변수만 채우고 나머지는 빈칸.
- 숫자만 적습니다(단위·°·"초" 같은 글자 넣지 말 것). 예: `52`, `95`, `0`.
- 칸 설명은 §6 표 참고. 다 적었으면 **다른 이름으로 저장** 시 형식은 **CSV(쉼표로 구분)** 유지.

> 엑셀 사용이 더 익숙하면 .xlsx로 작업해도 됩니다. 단 **열(헤더) 이름은 바꾸지 마세요** — IT 변환이 그
> 이름을 보고 처리합니다. 제출 시 .xlsx 그대로 주시면 IT가 CSV로 변환합니다.

## 5. 촬영조건(메타) 기입 — 층화(§8.9)

검증셋이 실제 촬영 조건을 반영하도록 아래도 함께 적습니다(각 영상 줄에).

| 칸 | 값 | 설명 |
|---|---|---|
| `viewpoint` | `sagittal`/`frontal`/`other` | 옆/앞/기타 — 굴곡각은 보통 `sagittal` |
| `occlusionLevel` | `none`/`partial`/`heavy` | 가림 정도 |
| `multiplePeople` | `Y`/`N` | 화면에 여러 명? |
| `clothing` | `light`/`heavy`/`ppe` | 복장(얇음/두꺼움/보호구) |
| `cameraHeight` | `low`/`eye`/`high` | 카메라 높이 |
| `workType` | 자유서술(비식별) | 작업유형(예: 조립, 운반) |

## 6. 엑셀 칸(열) 전체 설명

| 열 이름 | 의미 | 예시 |
|---|---|---|
| `caseId` | 사례 식별 번호(IT 안내값) | `case-001` |
| `videoRef` | 비식별 영상 참조(환자정보 금지) | `b2-001` |
| `clipFileName` | 원본 파일명(비식별 권장) | `b2-001-side.mp4` |
| `viewpoint` | 촬영 시점 | `sagittal` |
| `bodyPartFocus` | 평가 부위(측정 안내용) | `척추` |
| `clipDurationSec` | 클립 전체 길이(초) | `48` |
| `activeMinutesPerDay` | 이 작업의 1일 활동시간(분) — **작업조사값**(모르면 빈칸) | `360` |
| `trunkPeakDeg` | 체간 전굴 최대각(°) | `52` |
| `trunkOver45Sec` | 체간 45°↑ 누적시간(초) | `18` |
| `neckPeakDeg` | 목 굴곡 최대각(°) | `28` |
| `neckNonNeutralSec` | 목 비중립(>20°) 누적시간(초) | `12` |
| `kneeMinDeg` | 무릎 최소각(°) | `72` |
| `squatSec` | 쪼그림 누적시간(초) | `0` |
| `overheadSec` | 오버헤드 누적시간(초) | `0` |
| `repetitionCount` | 반복 횟수(클립 내) | `6` |
| `occlusionLevel` | 가림 | `partial` |
| `multiplePeople` | 다인원 | `Y` |
| `clothing` | 복장 | `heavy` |
| `cameraHeight` | 카메라 높이 | `eye` |
| `workType` | 작업유형 | `조립` |
| `annotator` | 평가자 코드(실명 대신) | `rater-A` |
| `annotatedDate` | 측정일 | `2026-06-22` |
| `notes` | 비고 | `측면 가림으로 무릎 불가` |

> `activeMinutesPerDay`는 "1일 중 그 작업을 실제로 한 시간"입니다(작업조사/면담값). 시간형 변수(누적시간)를
> 1일 노출로 환산할 때 IT가 사용합니다. 모르면 비워도 되며, 그 경우 시간형 변수는 검증에서 제외됩니다.

## 7. 품질 관리 — 평가자 간 교차측정(inter-rater)

- 전체의 **일부(예: 5~10개)** 영상은 **2명 이상이 서로 모르게 독립 측정**합니다(각자 다른 `annotator` 코드).
- 두 사람 값이 크게 다르면 그 변수는 "사람끼리도 잘 안 맞는" 변수이므로, AI 오차와 구분해 기록합니다.
- 측정 정의(§3)를 통일하기 위해 **첫 1~2개는 함께 측정**하며 기준을 맞추길 권장합니다.

## 8. 제출 & 이후(자동 처리)

1. 완성한 엑셀(.csv 또는 .xlsx)을 **IT 담당에게 전달**(비식별 확인). .xlsx면 IT가 CSV로 내보냅니다.
2. 이후는 IT가 처리합니다(아래 명령):
   ```bash
   # (1) 엑셀 CSV → annotations.json + manifest 스켈레톤 자동 변환(퍼-데이 환산 포함).
   #     --video-dir에 영상 폴더를 주면 manifest videoPath가 <폴더>/<파일명>으로 자동 채워짐(경로 수정 불필요).
   npm run video:annotations-from-csv -- --csv gold.csv --video-dir ./영상폴더 \
     --out-annotations annotations.json --out-manifest manifest.json
   # (2) (보통 생략) 다인원 영상에서 비주인공을 측정한 case만 manifest의 targetTrackId 지정.
   #     1인/주인공 영상은 빈칸이면 AI가 자동(dominant track)으로 선택.
   python services/pose-inference/validate_set.py --manifest manifest.json --output-dir out
   # (3) 사람 측정값 vs AI값 오차(§8.9) 리포트
   npm run video:validate-report -- --bundle out/validation_bundle.json --annotations annotations.json --out out/report.json
   ```
   - 변수별 허용오차(각도 ±10~15°, 시간 ±20%, 반복 ±15~20%) 충족 여부 판정 → 임계값 결정.
3. **검증 직접 대상 칸**: `trunkPeakDeg`·`trunkOver45Sec`·`neckNonNeutralSec`·`squatSec`·`overheadSec`(AI가 산출하는 변수).
   `neckPeakDeg`·`kneeMinDeg`·`repetitionCount`는 **참고·향후용**(현재 AI 비교 대상 아님 — kneeMinDeg는 쪼그림 구간 판단에 활용).
4. 변환 규칙(IT 참고, 스크립트가 자동 처리): `(누적초 / clipDurationSec) × activeMinutesPerDay = 분/일`
   (hours_per_day는 ÷60). 각도는 원값 그대로. `clipDurationSec`/`activeMinutesPerDay` 없으면 그 시간형 변수는 생략.

## 부록 B. 자주 묻는 질문

- **Q. 각도 기준선(수직)이 헷갈려요.** A. 항상 "중력 방향(천장으로 똑바로 위)"이 0° 기준입니다. 몸이 기운
  배경이어도 화면 세로가 수직입니다.
- **Q. 정면 영상인데 체간 전굴을 재도 되나요?** A. 전굴은 측면에서 정확합니다. 정면뿐이면 그 변수는
  비워두고 `notes`에 적으세요.
- **Q. 1일 활동시간(`activeMinutesPerDay`)을 제가 어떻게 아나요?** A. 작업조사/면담에서 나온 값입니다.
  모르면 비워두세요(IT가 manifest에서 채우거나, 시간형 변수는 제외됩니다).
- **Q. 숫자에 단위를 같이 써도 되나요?** A. 안 됩니다. 숫자만(예: `52`). 단위는 칸 정의에 이미 정해져 있습니다.
