# 통합 직업성 질환 평가 시스템 아키텍처 플랜

**작성일**: 2026-03-19
**버전**: 1.0

## Context

무릎 평가(React)과 MDDM 척추 평가(Vanilla JS)를 하나의 모듈형 시스템으로 결합.
향후 고관절, 어깨 등 확장 예정 → **플러그인 아키텍처** 채택.
배포: **웹(Vercel) + 데스크톱(Electron)** 동시 지원.
기술: **React 18 + Vite** 통일 (MDDM을 React로 마이그레이션).

## 2. 두 프로그램 비교 분석

### 공유 가능 (공통 코드)
| 영역 | 무릎 | 척추 (MDDM) |
|------|------|-------------|
| 인적사항 | ✅ 이름, 성별, 나이, 신장, 체중, BMI | ✅ 동일 |
| 진단정보 | ✅ 코드, 상병명, 부위 | ✅ 동일 구조 적용 가능 |
| 데이터 관리 | ✅ localStorage 저장/불러오기 | ✅ 동일 패턴 |
| Excel/PDF 내보내기 | ✅ SheetJS + html2pdf | ✅ SheetJS (PDF는 window.print) |
| 설정/테마 | ✅ CSS Variables, 다크모드 | ✅ CSS Variables |
| AI 분석 | ❌ 미구현 | ✅ Claude API 연동 |

### 모듈별 고유 (분리 필요)
| 영역 | 무릎 | 척추 (MDDM) |
|------|------|-------------|
| 노출 요인 입력 | 쪼그리기(분/일) + 보조요인 6종 체크 | G1-G11 자세 + F1-F4 보정계수 + 작업별 중량/빈도 |
| 직업력 모델 | 복수 직업 (시작일~종료일) + 프리셋 100+ | 단일 직업 (근무일수, 경력) |
| 계산 엔진 | 신체부담정도(4단계) × 근무기간 ÷ 연령보정 | F=b+m·L → 일일용량 → 평생용량 → 기준비교 |
| 평가/판정 | KLG 등급(0-4) + 업무수행능력(고/저) + 사유 | DWS2/법원/MDDM 3기준 비교 + 4단계 업무관련성 |
| 결과 출력 | 기여도 min~max% + 누적부담 충분/불충분 | MN·h 평생용량 + 위험도 게이지 |
| AI 프롬프트 | 무릎 전문가 system prompt (신규) | MDDM 전문가 system prompt (기존) |

### 3.2 AI API 호출 분기

AI 분석은 웹과 Electron에서 다르게 처리:

| 환경 | AI API 호출 경로 | API 키 위치 |
|------|-----------------|-------------|
| **웹 (Vercel)** | `fetch('/api/analyze')` → Vercel 서버리스 함수 | 서버 환경변수 (CLAUDE_API_KEY) |
| **Electron** | IPC → main process → 직접 Claude API 호출 | 앱 설정에서 사용자 입력 (로컬 저장) |

```javascript
// src/core/hooks/useAIAnalysis.js
async function callAI(prompt, systemPrompt, model) {
  if (window.electron) {
    // Electron: IPC를 통해 main process에서 직접 API 호출
    return await window.electron.analyzeAI({ prompt, systemPrompt, model });
  } else {
    // 웹: Vercel 서버리스 함수 경유
    const res = await fetch('/api/analyze', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ prompt, systemPrompt, model })
    });
    return await res.json();
  }
}
```

```javascript
// electron/main.js (AI 핸들러 추가)
ipcMain.handle('analyze-ai', async (_event, { prompt, systemPrompt, model }) => {
  const apiKey = store.get('claudeApiKey');
  const response = await fetch('https://api.anthropic.com/v1/messages', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': apiKey,
      'anthropic-version': '2023-06-01',
      'anthropic-beta': 'prompt-caching-2024-07-31'
    },
    body: JSON.stringify({
      model: model || 'claude-haiku-4-5-20251001',
      max_tokens: 2000,
      system: [{ type: 'text', text: systemPrompt, cache_control: { type: 'ephemeral' } }],
      messages: [{ role: 'user', content: prompt }]
    })
  });
  return await response.json();
});
```

## 9. 검증 방법

### 웹 버전
1. `npm run dev` → http://localhost:3000
2. 랜딩에서 "무릎" 선택 → 기존과 동일 동작 확인
3. 랜딩에서 "척추" 선택 → 기존 MDDM과 동일 계산 결과
4. AI 분석: 각 모듈별 다른 system prompt 호출 확인
5. `vercel deploy` → 프로덕션 배포

### Electron 버전
1. `npm run electron:dev` → 데스크톱 앱 실행
2. 설정에서 Claude API 키 입력 → AI 분석 동작 확인
3. 메뉴 → 평가 유형 전환 동작
4. `npm run electron:build:win64` → NSIS 설치파일 생성
5. 설치 후 실행 → 정상 동작 확인

- 무릎+척추 환자 혼합 저장/불러오기
- Excel 내보내기: 모듈별 고유 양식
- 다크모드 전환
